# 哪些可以删、哪些不能删（不影响项目）

下面这些**可以删**，删掉后不影响项目；下次用 Android Studio 打开或点“运行”时会自动重新生成。

---

## 可以删的（可再生成）

| 目录/文件 | 说明 |
|-----------|------|
| **`.gradle/`** | Gradle 缓存（任务、依赖哈希等），体积较大。删后下次同步会重新下载/生成。 |
| **`build/`** | 项目根目录的编译输出。删后下次编译会重新生成。 |
| **`app/build/`** | app 模块的编译输出（含 APK、中间文件）。删后下次编译会重新生成。 |
| **`.idea/`** | Android Studio 在本机的项目设置（运行配置、代码风格等）。删后重新打开项目会再生成，可能需重新选 SDK。 |
| **`local.properties`** | 本机 Android SDK 路径，每台电脑不同。删后重新打开项目会再生成（会提示选 SDK）。 |
| **`*.iml`** | 模块描述文件（如 `app.iml`）。删后重新打开项目会再生成。 |
| **`.externalNativeBuild/`** | 若存在：NDK 编译缓存。删后下次编译会再生成。 |
| **`.cxx/`** | 若存在：CMake/NDK 构建缓存。删后下次编译会再生成。 |
| **`captures/`** | 若存在：设备截图/录屏保存目录。可随意删。 |

**建议**：想省空间或“清干净再编译”时，删 **`.gradle/`** 和 **`build/`**（以及 **`app/build/`**）即可，其它一般不用管。

---

## 不能删的（删了项目会坏或无法运行）

| 目录/文件 | 说明 |
|-----------|------|
| **`app/src/`** | 全部源代码（Java、资源、清单）。 |
| **`app/build.gradle`** | app 模块的构建配置。 |
| **`app/proguard-rules.pro`** | 混淆规则（若用混淆）。 |
| **`build.gradle`** | 项目级构建配置。 |
| **`settings.gradle`** | 项目与模块列表。 |
| **`gradle/`** | 若存在：Gradle Wrapper（指定 Gradle 版本），删了可能无法在命令行构建。 |
| **`app/src/main/AndroidManifest.xml`** | 应用清单，必须保留。 |
| **`app/src/main/res/`** | 布局、字符串、图标等资源。 |
| **`README.md`、`push-map-to-emulator.ps1`、`.gitignore`** | 文档和脚本，删了不影响编译，但建议保留。 |

---

## 总结

- **可以放心删**：`.gradle/`、`build/`、`app/build/`、`.idea/`、`local.properties`、`*.iml`。
- **不要删**：`app/src/`、所有 `build.gradle` 和 `settings.gradle`、`AndroidManifest.xml`、`res/`。

删完可删的后，用 Android Studio 重新打开项目，等 Gradle 同步完成，再运行一次即可。
